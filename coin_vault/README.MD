# 个人金银币投资管理系统需求文档

**版本**：v1.0  
**日期**：2026-03-04  
**文档状态**：需求定义阶段

---

## 一、项目概述

### 1.1 背景
个人金银币投资者需要统一管理分散的藏品信息、交易记录及收益核算，解决当前依赖手工记账导致的克价成本模糊、盈亏计算困难、持仓状态不清等问题。

### 1.2 目标
构建一套单机运行的个人金银币投资管理系统，实现藏品全生命周期管理，核心关注克价成本追踪与投资收益分析。

### 1.3 范围
- 覆盖币种：金、银、铂、钯材质的投资币与纪念币
- 覆盖环节：采购入库、持仓管理、出售变现、收益核算
- 使用场景：个人本地使用，单用户操作

## 三、数据需求

| 字段分类 | 字段名称 | 数据类型 | 必填 | 说明 |
|:---|:---|:---|:---:|:---|

| 基础信息 | 名称 | 字符串 | 是 | 币种全称 |
| | 材质 | 枚举 | 是 | 金/银/铂/钯 |
| | 类型 | 枚举 | 是 | 投资币/纪念币/流通币 |

| | 发行年份 | 整数 | 是 | 铸造年份 |
| | 发行机构 | 字符串 | 否 | 中国人民银行等 |
| 规格 | 重量 | 小数 | 是 | 单位：克 |
| | 成色 | 字符串 | 是 | 如99.9% |
| | 面值 | 整数 | 否 | 法定货币面值，单位：元 |
| | 直径 | 小数 | 否 | 单位：毫米 |
| | 发行量 | 整数 | 否 | 限量发行数量 |

| | 证书编号 | 字符串 | 否 | 评级证书或原厂证书编号 |
| | 包装 | 字符串 | 否 | 原盒/封装/裸币 |

| | 年化收益率 | 小数 | 否 | 百分比 |

| | 更新时间 | 时间戳 | 是 | 最后修改时间 |
| | 图片路径 | 字符串 | 否 | 关联图片存储位置 |

### 3.2 数据规则

- 每枚藏品独立成条记录，即使同批次购买也分开展示
- 藏品ID全局唯一，不可重复
- 买入日期不得晚于卖出日期
- 重量、价格等数值字段保留3位小数精度
- 状态字段与是否已售字段保持逻辑一致

## 四、非功能需求

| 类别 | 需求描述 |
|:---|:---|
| 性能 | 支持1000条以上记录流畅查询，报表生成响应时间小于3秒 |
| 可靠性 | 数据操作后自动保存，异常退出不丢失已录入数据 |
| 易用性 | 界面简洁，核心操作（买入录入、卖出录入）三步内完成 |
| 可维护性 | 数据文件可独立备份，支持迁移至其他设备 |
| 安全性 | 支持可选的本地密码保护 |
| 兼容性 | 支持Windows/macOS/Linux系统运行 |

## 五、界面需求（简述）

### 5.1 主界面布局
- 顶部：全局搜索、快速统计卡片（持仓市值、累计盈亏、在库数量）
- 左侧：功能导航（藏品列表、买入录入、卖出录入、报表中心、设置）
- 主体：数据表格/录入表单/图表展示

### 5.2 核心界面
- **列表页**：表格展示藏品信息，支持排序、筛选、批量操作
- **买入录入页**：表单录入新购藏品，必填项高亮提示
- **卖出录入页**：选择持仓藏品，填写出售信息，实时预览盈亏
- **报表页**：图表展示持仓分布、收益趋势、克价对比

## 六、约束与假设

### 6.1 技术约束
- 单机运行，无需网络连接
- 数据本地存储，不支持多用户并发
- 币种限定为中国法定贵金属纪念币及主流投资币

### 6.2 业务假设
- 用户已具备基础金银币投资知识
- 金价数据可由用户手动录入或外部获取
- 盈亏计算以人民币为统一货币单位

## 七、附录

### 7.1 术语表

| 术语 | 说明 |
|:---|:---|
| 克价 | 每克贵金属的价格，用于横向比较不同重量藏品的成本 |
| 投资币 | 以投资为目的发行的标准化贵金属币，如熊猫普制币 |
| 纪念币 | 具有特定主题、限量发行的贵金属币，溢价较高 |
| 品相等级 | 第三方评级机构对藏品保存状态的评分，如MS70 |
| 溢价率 | 买入克价相对市场基准金价的超出比例 |
